課題1 ライブラリの作成
・mylib に移動して、make コマンドを実行して、mylib.a を生成する。


課題2 3目並べの作成
・03_3moku に移動して、goban.h 内でポート番号を指定する。
・make コマンドを実行して server と client を生成する。
・端末を2個開き、server と client を実行する。
・ゲームを楽しむ
  ・入力は a1 のように指定
  ・q を入力すると終了する

このプログラムには、いくつか問題がある。
・盤の表示がイマイチ。
・すでに置かれている石の位置にも石が打てる。
・勝敗判定がない。
・自分の石が x か o か判らなくなってしまう。


課題3 3目並べの改良
・以下の4機能を実装した server.mod client.mod を作成する。

手順：
・再利用するためにコピーする。
　　cp server.c server.mod.c
　　cp client.c client.mod.c
　　cp goban.c goban.mod.c
　　cp goban.h goban.mod.h
・Makefile を修正する。
・server.mod.c client.mod.c goban.mod.c goban.mod.h を修正する。

追加/修正する機能:
----- ----- ----- 
1. 枠を修正（追加）する。画面消去してから描画する。
                                  123
  +123+ 　　　　　　　　　　　　 +---+
  a...| 　　　　　　　　　　　　a|...|
  b...| 　→ このようにする →　b|...|
  c...| 　　　　　　　　　　　　c|...|
  +---+ 　　　　　　　　　　　　 +---+
----- ----- ----- 
2. 1手ごとに、勝敗を判定して、プログラムを終了させる。引き分けもありうる。
----- ----- ----- 
3. 入力した位置にすでに石が置かれていたら、入力をやり直す。
----- ----- ----- 
4. 実行時に、コマンドラインでホスト名やポート番号を指定する。
    > server.mod ポート番号
    > client.mod.c ホスト名 ポート番号

goban.mod.{c,h}について（改良のヒント）
  碁盤情報は構造体で管理する。たとえば、
    struct goban {
      char board[6][7]; /* 碁盤の情報 */
      char my_stone; /* 自分の石 */
      char peer_stone; /* 相手の石 */
      int sockd; /* ソケットのディスクリプタ */
    }
  goban_init()
    goban_plane[] の内容を初期化する。

  goban_show()
    escape sequence を使って画面を消去してから、盤を描画する。

  goban_peer_turn()
    石を置く盤上の座標を求める。

  goban_my_turn()
    おかしな入力ならば、入力をやり直す。
    石を置く盤上の座標を求める。
    入力した位置に すでに石が置かれていたら、入力をやり直す。
    石を盤上に配置する。
    goban_show() を呼び出して盤を表示する。
    write() に引数を与えて、クライアントへデータを送信する。
    勝敗メッセージを表示する。

  goban_check() 〔新設〕
    勝敗の確認をする。
    戻り値
      0  勝敗は未決定（ゲーム継続）
      1  相手の勝ち
      2  自分の勝ち
      3  引き分け
    勝利条件に対し、正しく判定できることを確認すること！
      ・上・中・下の行、左・中・右の列、左上→右下、右上→左下 の各々で3つ
        並んだときに、勝ちを判定できるか。
      ・同様に、相手の勝ちを判定できるか。
      ・引き分けも、確認する（9石を打った後のみ）。
